### **🔍 核心背景与痛点分析**

**传统视频剪辑痛点**
- **效率低下**：长视频（如1.5小时直播）人工剪辑耗时，需逐段筛选精华内容
- **AI判断偏差**：现有AI工具仅基于逐字稿剪辑，常出现"AI认为的精华≠人类判断的精华"
- **场景适配不足**：办公场景（如用户访谈剪辑）需要快速提取原声片段，传统工具操作复杂

### **🛠️ 技术方案与工作流设计**

#### **一、核心技术组件**

| 组件 | 功能说明 | 关键作用 |
| :--- | :--- | :--- |
| **飞书妙记** | 提供带时间戳的逐字稿与视频源文件 | 作为剪辑内容的原始数据来源 |
| **飞书CLI** | 开源命令行工具，支持读取妙记内容与下载视频源文件 | 实现视频数据的程序化获取 |
| **Coding Agent** | 以Claude Code为例，处理自然语言指令并生成剪辑逻辑 | 承担"大脑"角色，理解用户需求并执行剪辑决策 |
| **FFmpeg** | 视频处理工具，支持按时间戳精确切割视频 | 执行具体的视频剪辑操作 |

#### **二、四步式剪辑工作流**
1. **内容分析阶段**  
   将飞书妙记内容发送给Coding Agent，指令其按"嘉宾演示场景"维度总结视频结构（如"总结各嘉宾演示的使用场景"）

2. **逐字稿导出与标记**  
   - 从飞书妙记导出含**说话人+时间戳**的逐字稿至飞书文档  
   - 用户在文档中手动高亮需剪辑的片段，通过评论补充调整需求（如"补充XX句"、"去掉XX段"）

3. **AI协作对齐**  
   - Coding Agent根据用户标记的高亮片段，在文档中生成黄色/橙色等不同颜色的标记  
   - 用户与AI通过文档评论进行多轮调整，确保剪辑范围精准匹配预期

4. **自动化剪辑执行**  
   - Coding Agent调用飞书CLI下载视频源文件  
   - 解析文档中的时间戳标记，调用FFmpeg完成视频片段切割  
   - 输出多个独立的高光视频（如2分钟/段的场景化切片）
