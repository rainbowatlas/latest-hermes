### Claude Blog：为并行 Agent 重新设计 Claude Code 桌面端
Anthropic 重新设计了 Claude Code 桌面应用，以适配当前的 agentic 编程工作流：
- **新侧边栏**：集中管理多个活跃会话，按状态/项目/环境筛选，会话完成自动归档
- **侧边聊天**（⌘+;）：从主线程分叉对话，不污染主线程上下文
- **并行工作**：在多个 repo 同时启动任务，在结果到来时切换
- **应用内工具集成**：集成终端、文件编辑器、更快的 diff 查看器、扩展预览
- **拖拽布局**：所有面板可自由排列成适合你工作方式的网格

核心洞察："对于许多开发者来说，agentic 工作的形态已经改变了。你不是输入一个提示然后等待。你是在一个 repo 启动重构，在另一个 repo 修复 bug，在第三个 repo 写测试，在结果到来时检查，在偏离时引导，在发布前审查 diff。新应用就是为了这种工作状态而构建的。"[博文链接](https://claude.com/blog/claude-code-desktop-redesign)

### AI & I by Every — "The AI Sandwich: Where Humans Excel in an AI World"

**嘉宾**：Kieran Klaassen，Quora GM，Compound Engineering 框架创建者

**核心观点**："人类是三明治的面包，AI 是中间的馅料。"Klaassen 认为在任何流程中，人类在开头（设定问题框架）和结尾（判断输出是否'感觉对了'）不可替代，中间的执行层可以被 AI 很好地自动化。

他介绍了 Compound Engineering 的四步框架：Plan → Work → Review → Compound。其中最后一步——将经验反馈到 repo 中，让 Agent 不再重复犯错——是让一个工程师能像五人团队一样交付的关键。

**最精辟的一句话**："如果你想让成果属于你自己，你不能完全自动化一切。就像艺术一样，它需要来自你，或者以某种方式与你相连。"

这场对话对任何在思考"AI 能写代码之后，我还能做什么"的人都是必读。
