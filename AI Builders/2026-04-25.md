## 📝 官方博客

### Anthropic Engineering: Scaling Managed Agents — 将大脑与双手解耦

Anthropic 发布了一篇深度工程文章，介绍 Claude Managed Agents 的架构设计理念。核心观点：

**架构哲学**：借鉴操作系统的设计模式，将 agent 的三个组件虚拟化——session（append-only 日志）、harness（调用 Claude 并路由工具调用的循环）、sandbox（代码执行环境）。这样每个组件的实现可以独立替换而不影响其他部分，就像 read() 命令不关心底层是 1970 年代的磁盘还是现代 SSD。

**从宠物到牲畜**：最初将所有组件放在单个容器中，遇到了经典的"宠物"问题——容器挂了 session 就丢了，调试无响应会话极其困难。解决方案是将组件解耦，让每个部分都可以独立扩缩容和替换。

**Harness 会过时**：Harness 编码了对"Claude 不能做什么"的假设，但这些假设会随模型改进而过时。例如之前发现 Sonnet 4.5 有"上下文焦虑"（接近 context limit 时过早结束任务），所以加了 context resets。但同样的 harness 用在 Opus 4.5 上时，这个行为消失了，resets 变成了累赘。Managed Agents 的设计就是要让这些实现细节可以自由演进。

**"为尚未想到的程序设计"**：这是 computing 的老问题。Managed Agents 通过抽象接口来解决——对接口的形状有意见，对背后运行的实现没有意见。
